Upgrade, Backup, and Restore

Part of a good system design also includes regular backups to restore to if necessary.

Some of the benefits of VMs or virtualized machines/servers in the cloud are the abilities to add processing power, RAM, and storage and to make backups (snapshots), and to restore from snapshots.

Each person individually needs to do the following:

  1. Add more RAM to your EC2 instance.
  2. Start your EC2 instance and connect to it.
  3. Using the AWS Console in your browser, create a snapshot. Now we have a backup.
  4. Now, let's simulate a hacker creating a file on your server: Using the connected terminal, create or download a file (ex: touch hackerfile.sh).
  5. Using the AWS Console, restore the snapshot:
    1. Stop the EC2 instance.
    2. Detach its current volume.
    3. From the snapshot, create a volume in the same availability zone as the previously current volume.
    4. Attach the new volume to the EC2 instance.
    5. Start the EC2 instance.
  6. Connect to your EC2 instance and verify that the file was removed.

Help your team. Take screenshots or make a video to turn in.
