ITM 101 Final Project

(Min-Max Inventory System)

Scenario

Many companies have a storeroom or warehouse where they temporarily store items that they will sell or use to manufacture products. The items in a storeroom or warehouse are called inventory.

An inventory system is a computer system used by employees to track items in a storeroom or warehouse. The employees must track the inventory so that they know what items are available and what items need to be ordered from a manufacturer. Companies want to store the right amount of items in their warehouse. They want to store enough of all items so that they can continue to manufacture products or sell to customers. But storing more of an item than the company needs is costly to the company.

A min-max inventory system is a simple inventory system used by businesses to manage their inventory levels efficiently. In a min-max inventory system, a company chooses a minimum (min) and maximum (max) quantity for each item in inventory. When the current quantity of an item drops to the minimum threshold, the system automatically generates a reorder to bring the quantity back up to the maximum level.

Your boss has asked you to create and deploy a min-max inventory system that uses a NoSQL database in the Internet Cloud.

The JavaScript code must be contained in a function that a browser will execute after the HTML document is loaded. The JavaScript code must use the Firebase and Firestore Web modular API functions to add the supplier data into a collection named "suppliers". After adding each supplier, the JavaScript code must retrieve the document ID for the supplier and display the document ID and supplier name in a div in the HTML document.
