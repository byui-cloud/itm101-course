ITM 101 Final Project

(Min-Max Inventory System)

Part 2

Scenario

The Min-Max Inventory System that you created and deployed last week is using development security rules. This means that anyone can view, edit, and delete the data in your Firestore database. Your boss has asked you to change the Min-Max Inventory System so that it authenticates users and only allows authenticated users with the correct permissions to view, edit, and delete the data.

Steps

Do the following:

  1. Use ChatGPT to learn about Google Firebase authentication and Firestore security rules. Start a new chat with ChatGPT, and give the following prompts to ChatGPT. Use ChatGPT as a learning assistant. Read it’s answers, and ask it other questions that will help you understand Firebase authentication and Firestore security rules.
  2. Using your browser, download the filestore.rules file and save it into the the min_max_inv folder that you created last week.
  3. Using your browser, download the following two files and save them into the min_max_inv/public folder that the firebase init hosting command created last week:
    1. index.html
    2. min_max_inv.js
  4. Using the Firebase Console in your browser, look in the "Project settings" for your min-max-inventory project and copy firebaseConfig dictionary. The dictionary will look similar to this: 
    // Your web app's Firebase configuration
const firebaseConfig = {
  apiKey: "a_bunch_of_characters_here",
  authDomain: "min-max-inventory-hex.firebaseapp.com",
  projectId: "min-max-inventory-hex",
  storageBucket: "min-max-inventory-hex.appspot.com",
  messagingSenderId: "lots_of_digits",
  appId: "more_characters_here"
};

    You should not copy the sample firebaseConfig shown above. Instead, you must copy the firebaseConfig from your min-max-inventory project in the Firebase Console.

  5. Open VS Code and in the "File" menu, click "Open Folder..." and open the min_max_inv folder that you created last week.
  6. Using VS Code, open the min_max_inv.js file that you downloaded three steps ago. At about line 98, you will find these three lines of code: 
      /* Copy and paste your project's firebaseConfig here. */
  const firebaseConfig = {
  };

    Replace those three lines of code with the firebaseConfig code that you copied from the Firebase Console in the two steps ago.

  7. In VS Code, open a new terminal and execute the following command: 
    firebase deploy --only "firestore,hosting"
  8. Look in the VS Code terminal below the fireabase deploy command for the URL where the command sent your files. The URL will be similar to this: 
    https://min-max-inventory-digits.web.app

    Copy the URL from the VS Code terminal and open it in a new tab in your browser.

  9. Using your browser, test your Min-Max Inventory system thoroughly by doing the following:
    1. Look at each of the five tabs and verify that the each show the correct data from your Firestore database.
    2. In the Receiving tab, enter numbers in the text boxes, press the Submit button, and verify that the corresponding Current Quantities change as they should.
    3. In the Outgoing tab, enter numbers in the text boxes, press the Submit button, and verify that the corresponding Current Quantities change as they should.
  13. Using your browser in the Firebase console, enable authentication.
  14. If you’re curious, you could ask your teacher or ChatGPT what is the purpose of each of the files in a web app.
